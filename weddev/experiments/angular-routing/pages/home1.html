Experiments

An application which demonstrates the use of angular controllers that are used in the navigation pages yet the source code remains with the main page

In this experiment, I worked with the concept of angular-routing along with the controller functions by creating 3 seperate files about.html, home.html and notify.html.
The pages are then linked with the routing and shown in the container of the main page using ng-view attribute.
There is a function defined on the button available in the notify page. On clicking notify me the email is registered and shown in the table below. This feature can be handy when we make changes to cache or database with in the same page irrespective of other pages.

Code Snippet


The code snippet shows how a application is configured for each relative url. There are different cases specified for different urls and the default case is mentioned in otherwise clause. The controller registerController is defined only for the notify.html page using the attribute controller. A method addEmail() is then defined to add the emailid to the registeredusers array and to display on the table below.
